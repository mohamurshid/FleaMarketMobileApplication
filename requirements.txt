Functional Requirements
1. User Management
•	The system shall allow students to register and log in using their Strathmore credentials (e.g., student email).
•	The system shall allow users to view and update their personal profiles.
•	The system shall allow the admin to:
Approve or reject user registrations.
Deactivate or remove users violating policies.

2. Item Listings
•	The system shall allow sellers to create new listings for items (for sale or auction).
•	The system shall allow sellers to:
o	Add details such as title, description, category, price, condition, and images.
o	Choose between a fixed price sale or an auction format.
•	The system shall allow the admin to:
o	Review and approve all new listings before they appear publicly.
o	Remove or suspend any listing that violates marketplace rules.
•	The system shall allow buyers to view approved listings filtered by categories, price, or keywords.
3. Auction Functionality
•	The system shall allow sellers to set:
o	Starting bid, bidding end time, and minimum increment value.
•	The system shall allow buyers to:
o	Place bids on active auction items.
o	View current highest bid and time remaining.
•	The system shall automatically:
o	Update the current highest bid in real-time.
o	Notify the winning bidder and seller when the auction ends.
4. Buying and Orders
•	The system shall allow buyers to:
o	Buy items instantly (if fixed price listing).
o	Confirm order and payment (integration can be manual or future).
•	The system shall allow sellers to:
o	Receive order notifications for items purchased or won in auction.
o	Mark orders as completed.
5. Search and Filtering
•	The system shall allow all users to search for items using keywords.
•	The system shall allow filtering by:
o	Category
o	Price range
o	Item type (sale or auction)
o	Condition (new/used)
6. Notifications
•	The system shall notify users via in-app alerts or email when:
o	Their listing is approved/rejected by admin.
o	They receive a new bid on their auction.
o	Their bid is outbid.
o	They win an auction.
o	Their item is sold or purchased.
7. Admin Management
•	The admin shall be able to:
o	Manage users (approve, suspend, or delete).
o	Approve or remove listings.
o	View platform activity logs (optional).
8. System Requirements
•	The system shall ensure that only approved users can list or buy items.
•	The system shall prevent bidding after auction end time.
•	The system shall prevent editing of items after listing approval (except by admin).
•	The system shall maintain a secure login and session management for all users.
